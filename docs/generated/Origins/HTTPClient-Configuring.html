Configuring an HTTP Client Origin

Configure an HTTP Client origin to read JSON data from a streaming HTTP resource URL.

  1. On the HTTP tab, configure the following properties:
    HTTP Property Description
    Data Format The HTTP Client reads JSON data.
    Resource URL URL where the data resides.
    Request Timeout Amount of time the origin waits for a response from the server.
    Mode Processing mode:
    • Streaming - Maintains a connection and processes data as it becomes available.
    • Polling - Connects periodically to check for data.
    Polling Interval (ms) Number of milliseconds the origin waits before checking for data. Used in the polling mode only.
    Use OAuth Determines whether OAuth credentials are used to connect to the server.
    Batch Size Maximum number of JSON objects to include in a batch and send through the pipeline at one time.
    Batch Wait Time Maximum amount of time to wait before sending a batch.
  2. When using OAuth, on the OAuth tab, configure the following properties:
    OAuth Property Description
    Consumer Key OAuth consumer key.
    Consumer Secret OAuth consumer secret.
    Token OAuth consumer token.
    Token Secret OAuth token secret.
  3. On the JSON tab, configure the following properties:
    JSON Property Description
    JSON Content Type of JSON content to be processed:
    • JSON Array of Objects - Use to process data in polling mode.
    • Multiple JSON Objects - Use to process data in streaming or polling mode.
    Entity Delimeter Delimiter to indicate the end of a JSON record, such as \n or \r. Used when processing multiple JSON objects.
