RPC

The RPC origin reads data passed from an RPC destination over the WAN. Use the RPC origin as part of an RPC destination pipeline that reads data from an RPC destination in the origin pipeline.

When you configure the RPC origin, you define the port that the origin listens to for data. The port number must be one of the <host>:<port> combinations defined in the RPC destination that provides the data.

Similarly, the RPC ID in the RPC origin must match the RPC ID used in the RPC destination.

You can configure the origin to use HTTPS to securely transfer data.

RPC Pipelines

Enabling SSL

You can configure the RPC origin to use SSL. To use SSL you need a valid keystore file and password.

To enable security:
  1. Save the keystore file in the Data Collector resources directory: <SDCinstalldir>/resources.
  2. In the RPC origin, enter the filename and password.

Configuring an RPC Origin

Configure an RPC origin to process data from an RPC destination.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the RPC tab, configure the following properties:
    RPC Property Description
    RPC Listening Port Port number to listen to for data. Must match one of the port numbers associated with the RPC destination that provides the data.
    RPC ID User-defined ID. Must match the RPC ID defined in the RPC destination.
    Batch Wait Time (ms) Number of milliseconds to wait before sending a partial or empty batch.
    SSL Enabled Enables the secure transfer of data over the WAN using SSL.
    Certificate Keystore File Keystore file for SSL. Must be stored in the Data Collector resources directory: <SDCinstalldir>/resources.
    Keystore Password Password for the keystore file.
