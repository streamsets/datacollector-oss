Configuring a Pipeline

Configure a pipeline to define the stream of data. After you configure the pipeline, you can start the pipeline.

A pipeline can include the following components:
  • A single origin stage
  • Multiple processor stages
  • Multiple destination stages
  1. From the Home page or Getting Started page, click Create New Pipeline.
    If you are in the Data Collector console, use the Toggle Library icon to display the pipeline library, and click the Create New Pipeline icon.
  2. In the New Pipeline window, enter a pipeline name and optional description, and click Save.
    The pipeline canvas displays pipeline name and an error icon. The error icon indicates that you need to configure error handling for the pipeline. The Properties panel displays the pipeline properties.
  3. If this is the first pipeline for the Data Collector, click Create Pipeline.
    If you have created other pipelines, if necessary, use the Toggle Library icon to display the pipeline library, and click the Create New Pipeline icon.
  4. In the New Pipeline window, enter a pipeline name and optional description, and click Save.
    The pipeline canvas displays the new name. The Properties panel displays the pipeline properties.
  5. In the Properties panel, on the General tab, configure the following properties:
    Pipeline Property Description
    Description Optional description of the pipeline.
    Execution Mode Execution mode of the pipeline:
    • Standalone - A single Data Collector process runs the pipeline.
    • Cluster - Additional Data Collector workers are spawned as needed to process data. Limits the origins that you can use in the pipeline.
    Delivery Guarantee Determines how the Data Collector handles data after an unexpected event causes the pipeline to stop running:
    • At Least Once - Ensures all data is processed and written to the destination. Might result in duplicate rows.
    • At Most Once - Ensures that data is not reprocessed to prevent writing duplicate data to the destination. Might result in missing rows.

    Default is At Least Once.
    Memory Limit Maximum amount of memory for the pipeline to use. You can enter a numeric value or edit the default expression to use a percentage of the Java heap size defined for the Data Collector.
    Default is 65% of the Data Collector Java heap size:
     ${jvm:maxMemoryMB() * 0.65}
    Memory Exceeded Action to take when the pipeline memory reaches the memory limit:
    • Log - Logs a message in the pipeline history.
    • Log and Alert - Logs a message and triggers an alert that displays in monitor mode and sends an alert email to any provided email addresses.
    • Log, Alert and Stop Pipeline - Logs a message, triggers an alert that displays in monitor mode and sends an alert email to any provided email addresses. Stops the pipeline. This option is not supported for cluster mode at this time.
  6. To define pipeline constants, on the Constants tab, click the Add icon and define the name and the value for each constant.
  7. Click the Error Records tab and configure the following error handling option:
    Error Records Property Description
    Error Record Handling Determines how to handle records that cannot be processed as expected. Use one of the following options:
    • Discard - Discards error records.
    • Write to File - Writes error records to a file in the specified directory.
    • Write to Kafka - Writes error records to the specified Kafka cluster.

    For cluster mode, use Discard or Write to Kafka. Writing to file is not supported at this time.

  8. When writing error records to file, click the Error Records - Write to File tab and configure the following properties:
    Write to File Property Description
    Directory Local directory for error record files.
    File Prefix Prefix used for error record files. Use to differentiate error record files from other files in the directory.

    Uses the prefix sdc-${sdc:id()} by default. The prefix evaluates to sdc-<Data Collector ID>. This provides default differentiation in case several Data Collectors write to the same directory.

    The Data Collector ID is stored in the following file: <SDCinstalldir>/data/sdc.id.
    File Wait Time (secs) Number of seconds the Data Collector waits for error records. After that time, it creates a new error record file.

    You can enter a number of seconds or use the default expression to enter the time in minutes.

    Max File Size (MB) Maximum size for error files. Exceeding this size creates a new error file.

    Use 0 to write to avoid using this property.
  9. When writing error records to Kafka, click the Error Records - Write to Kafka tab and configure the following properties:
    Write to Kafka Property Description
    Broker URI Connection string for the Kafka broker. Use the following format: <host>:<port>.

    To ensure a connection, enter a comma-separated list of additional broker URI.
    Runtime Topic Resolution Evaluates an expression at runtime to determine the topic to use for each record.
    Topic Expression Expression used to determine where each record is written. Use an expression that evaluates to a topic name.

    For runtime topic resolution only.

    Topic White List List of valid topic names to write to. Use to avoid writing to invalid topics. Records that resolve to invalid topic names are passed the stage for error handling.

    Use an asterisk (*) to allow writing to any topic name. By default, all topic names are valid.

    For runtime topic resolution only.

    Topic Kafka topic to use.

    Not available when using runtime topic resolution.
    Partition Strategy Strategy to use to write to partitions:
    • Round Robin - Takes turns writing to different partitions.
    • Random - Writes to partitions randomly.
    • Expression - Uses an expression to write data to different partitions.
    Partition Expression Expression to use to write to partitions.

    Define the expression to evaluate to the partition where you want each record written. Partition numbers start with 0.

    Optionally, click Ctrl + Space Bar for help with creating the expression.

    For the expression partition strategy only.

    One Message per Batch For each batch, writes the records to each partition as a single message.
    Kafka Configuration Additional Kafka properties to use. Click the Add icon and define the Kafka property name and value.

    Use the property names and values as expected by Kafka. Do not use the broker.list property.

    Several properties are defined by default. You can edit or delete the properties.

  10. When using the cluster execution mode, click the Cluster tab and configure the following properties:
    Cluster Property Description
    Worker Memory (MB) Maximum amount of memory allocated to each Data Collector worker in the cluster.

    Default is 1024 MB.
    Worker Java Options Additional Java properties for the pipeline. Separate properties with a space.

    The following properties are set by default.

    • XX:PermSize is set to 128 MB and XX:MaxPermSize is set to 256 MB. This defines the PermGen size for the pipeline.
    • Dlog4j.debug enables debug logging for log4j.

    Changing the default properties is not recommended.

    You can add any valid Java property.

    Kerberos Authentication

    Uses Kerberos credentials to connect to the YARN cluster.

    When selected, uses the Kerberos principal and keytab defined in the Data Collector configuration file.

    Launcher Env Configuration

    Additional configuration properties for the cluster launcher. Click the Add icon and define the property name and value.

  11. Use the Stage Library to add an origin stage. In the Properties panel, configure the stage properties.
    For configuration details about origin stages, see Origins.
  12. Use the Stage Library to add the next stage that you want to use, connect the origin to the new stage, and configure the new stage.
    For configuration details about processors, see Processors.

    For configuration details about destinations, see Destinations.

  13. Add additional stages as necessary.
  14. At any point, you can use the Preview icon to preview data to help .configure the pipeline. For more information, see Data Preview.
  15. Optionally, you can create metric or data alerts to track details about a pipeline run and create threshold alerts. For more information, see Alerts.
  16. When the pipeline is complete, use the Start icon to run the pipeline.
When the Data Collector starts the pipeline, the Monitor console displays real-time statistics for the pipeline. For more information about monitoring, see Pipeline Monitoring.
