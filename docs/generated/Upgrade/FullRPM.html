Upgrade a Full Installation from the RPM Package

When you upgrade a full installation from the RPM package, the new version uses the default directories to store configuration, data, log, and resource files. If the previous version used the default directories, the new version has access to the pipeline, log, and resource files created in the previous version.

If the previous version used customized values for the directory environment variables, you must make the same customizations in the new version so that the new version can access the same pipeline, log, and resource files.

To upgrade a full installation from the RPM package, perform the following steps:

  1. Shut down the previous version.
  2. Install the new version.
  3. Manually update the environment configuration file.
  4. Manually update the configuration files.
  5. Start the new version of Data Collector.

Step 1. Shut Down the Previous Version

Shut down the previous version of Data Collector.

To use the command line for shutdown, use the Data Collector process ID in the following command:

kill -3 <process ID>

To use the Data Collector console for shutdown:

  1. Click Administration > Shut Down.
  2. When a confirmation dialog box appears, click Yes.

Step 2. Install the New Version

Install the new version of the RPM package.

  1. Use the following URL to download the Data Collector RPM package from the StreamSets website: https://streamsets.com/opensource.
  2. Use the following command to install the Data Collector RPM package in a different directory than the previous version: 
    yum localinstall streamsets-datacollector-<version>-1.noarch.rpm
    For example, to install version 1.5.0.0, use the following command:
    yum localinstall streamsets-datacollector-1.5.0.0-1.noarch.rpm

Step 3. Update the Environment Configuration File

Each RPM installation uses the same default values as the previous version for all of the directory environment variables. If the previous version used the default values, the new version is configured to use the same working directories.

If the previous version used customized values for the directory environment variables, you must make the same customizations in the new version. The new version must use the same data, log, and resource directories as the previous version.

  1. In the new version of Data Collector, open the $SDC_DIST/libexec/sdcd-env.sh file.
  2. Manually update the new environment configuration file with any customized property values that you defined in the previous version.

Data Collector Environment Configuration

Step 4. Update the Configuration Files

A new Data Collector version can include new properties and configuration files required for Data Collector to start or function properly. The configuration files are stored in the $SDC_CONF directory, including the sdc.properties file and the related realm properties files, keystore files, LDAP files, and the logj4 properties file.

When you install the new RPM package, the configuration files are written to the same default directory as the previous version, /etc/sdc. The new versions of the configuration files are renamed with the following extension: .rpmnew. For example, the new version of the Data Collector configuration file is renamed to sdc.properties.rpmnew.

To update the configuration files, you must rename the previous and new versions of the files and then update the new files with any customized property values defined in the previous version.

Note: If the previous version used a customized value for $SDC_CONF, the new configuration files are written to a different directory than the previous version, and so do not require the .rpmnew file extension. In this case, you do not rename the configuration files, but must update the new files with any customized values defined in the previous version.
  1. In the working $SDC_CONF directory, /etc/sdc by default, rename all previous configuration files with the following extension: .old.
  2. Remove the following extension from all new configuration files: .rpmnew.
  3. Compare the previous and new versions of the configuration files, and update the new files as needed with the same customized property values.

Step 5. Start the New Version of Data Collector

Use the following command to start the new version of Data Collector:
service sdc start
