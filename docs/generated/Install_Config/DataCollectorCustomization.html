Data Collector Customization

You can customize Data Collector to define properties such as the host name and port number, the HTTP protocol to use, and account information for email alerts.

HTTP Protocols

You can configure the Data Collector to use HTTP or HTTPS. By default the Data Collector uses HTTP.

When you configure the Data Collector to use HTTPS, you have the Data Collector redirect HTTP requests to HTTPS or ignore them.

HTTPS requires an SSL Certificate. You can generate a certificate that is self-signed or signed by a certifying authority. Web browsers generally issue a warning for self-signed certificates.

When you generate a SSL Certificate, use the following files:
keystore file
A file that contains the private key and self-signed certificates for the web server. You can create your own keystore file or use the keystore file provided by the Data Collector.
Store the keystore file in the following location:
<SDCinstalldir>/etc
The default keystore file is sdc-keystore.jks.
keystore password file
A file that contains the password to open the Java keystore file. You can create your own keystore password file or use the file provided by the Data Collector.
Store the keystore password file in the following location:
<SDCinstalldir>/etc
The default keystore file is sdc-keystore-password.txt.

Generating a Self-Signed Certificate

You can use the Java keytool command to generate a self-signed certificate.

  1. From the command line, navigate to the following directory: <SDCinstalldir>/etc.
  2. Enter the following command: 
    $ keytool -genkey -alias sdc -keyalg RSA -keystore sdc-keystore.jks -storepass <password> -keypass <password>
    Note: Use the same password for the -storepass and -keypass parameters.
  3. When keytool requests your first and last name, enter a fully-qualified host name as follows: 
    <SDC host name>:<domain name>

    For example: mymachine.streamsets.com.

  4. When keytool requests the name of your organizational unit, enter the name.
  5. When keytool requests the name of your organization, enter the name.
  6. When keytool requests the name of your city or locality, enter the name.
  7. When keytool requests the name of your state or province, enter the name.
  8. When keytool requests the two character country code, enter the code.
  9. When keytool requests verification that the information is correct, enter yes if it is correct.
    The tool generates a self-signed certificate in the form of the sdc-keystore.jks file.
  10. Enter the keystore password in the keystore password file: <SDCinstalldir>/etc/sdc-keystore-password.txt.
  11. Save a copy of the sdc-keystore.jks and sdc-keystore-password.txt files.

Generating an Authority-Signed Certificate

You can use the Java keytool command to request a certificate from a certifying authority.

  1. From the command line, navigate to the following directory: <SDCinstalldir>/etc.
  2. Enter the following command: 
    $ keytool -certreq -v -alias sdc -keyalg RSA -keystore sdc-keystore.jks
  3. When keystore requests the keystore password, enter the password.
  4. When keystore generates a new certificate request, send the request to the certifying authority.
  5. When the certifying authority responds to the request, use the following command to verify the results: 
    $ keytool -printcert -file <certificate-reply-file>
  6. To import the response to the sdc-keystore.jks file, use the following command and enter the keystore password when requested: 
    $ keytool -v -import -trustcacerts -keystore sdc-keystore.jks -file certificate-reply-file -alias sdc
  7. To verify the imported certificate, use the following command and enter the keystore password when requested: 
    $ keytool -v -list -keystore sdc-keystore.jks
  8. Enter the keystore password in the keystore password file: <SDCinstalldir>/etc/sdc-keystore-password.txt.
  9. Save a copy of the sdc-keystore.jks and sdc-keystore-password.txt files.

Customizing the Data Collector

You can edit properties in the Data Collector properties file to customize the Data Collector.

Use a text editor to edit the Data Collector properties file. After you edit the file, restart the Data Collector to enable the changes.

The Data Collector properties file, sdc.properties, is located in the following directory:
<SDCinstalldir>/etc/sdc.properties
The Data Collector properties file includes the following general properties:
General Property Description
http.port Port number to use for the Data Collector console.

Default is 18630.
https.port Secure port number for the Data Collector console. Any number besides 0 enables the secure port number.

If you use both port properties, the http port bounces to the https port.
https.keystore.path Keystore path and file name.

By default, the file name is sdc-keystore.jks and expected in the same directory as the Data Collector properties file.
https.keystore.password File that contains the password to the keystore file.

By default, the file name is sdc-keystore-password.txt and expected in the same directory as the Data Collector properties file.
http.authentication HTTP authentication. Use none, digest, or form.

Default is form.
http.digest.realm Realm used by digest authentication.

Default is local-realm.
preview.maxBatchSize Maximum number of records used to preview data.

Default is 10.
preview.maxBatches Maximum number of batches used to preview data.

Default is 10.
production.maxBatchSize Maximum number of records included in a batch when the pipeline runs.

Default is 1000.
production.maxErrorFileSize Maximum error file size.

Default is 1024 MB.

production.maxBackupIndex Maximum number of error files the pipeline can create. When the error file count exceeds this number, older files are removed.

Enter a positive integer. If you enter 0, no additional files are created and the error file is truncated when it reaches the maximum size.

Default is 5.
production.maxErrorRecordsPerStage Maximum number of error records to save in memory for each stage to display during monitoring. When the limit is reached, older error records are discarded.

Default is 100.

production.maxPipelineErrors Maximum number of pipeline errors to save in memory to display during monitoring. When the limit is reached, older errors are discarded.

Default is 100.

max.logtail.concurrent.requests Maximum number of external processes allowed to access the Data Collector log file at the same time through REST API calls.

Default is 5.
ui.help.base.url Base URL for the local help project.

Do not change this value.
ui.hosted.help.base.url Base URL for the help project hosted at the StreamSets website.

Do not change this value.
ui.refresh.interval.ms Interval in milliseconds that the Data Collector waits before refreshing the Data Collector console.

Default is 2000.
ui.jvmMetrics.refresh.interval.ms Interval in milliseconds that the JVM metrics are refreshed.

Default is 4000.
ui.enable.usage.data.collection Allows anonymous Google Analytics information to be sent StreamSets.

Default is true.

The Data Collector properties file also includes the following properties for sending email alerts:
Email Alert Properties Description
mail.transport.protocol Use smtp or smtps.

Default is smtp.
mail.smtp.host SMTP host name.

Default is localhost.
mail.smtp.port SMTP port number.

Default is 25.
mail.smtp.auth Whether the SMTP host uses authentication. Use true or false.

Default is false.
mail.smtp.starttls.enable Whether the SMTP host uses STARTTLS encryption. Use true or false.

Default is false.

mail.smtps.host SMTPS host name.

Default is localhost.
mail.smtps.port SMTPS port number.

Default is 25.
mail.smtps.auth Whether the SMTPS host uses authentication. Use true or false.

Default is false.
xmail.username User name for the email account to send alert email.
xmail.password Password for the email account to send alert email.
xmail.from.address Email address to send alert email.
