Additional Drivers

You can install additional drivers for stages.

Before you use the following stages, you need to install drivers for the implementation that you want to use:
  • JDBC Consumer origin
  • JMS Consumer origin
  • Oracle CDC Client origin
  • JDBC Lookup processor
  • JDBC Tee processor
  • JDBC Producer destination

For example, to use the JDBC Consumer or JDBC Producer with Oracle, install the Oracle JDBC drivers.

Additional drivers for other stages are optional. For example, you can install custom classes for the JavaScript or Jython Evaluators.

Installing Additional Drivers

To install additional drivers for a stage, perform the following steps:
  1. Create a local directory external to the Data Collector installation directory for the drivers. Use an external directory to enable use of the drivers after Data Collector upgrades.
    For example, if you installed Data Collector in the following directory: 
    /opt/sdc/
    you might create the driver directory at: 
    /opt/sdc-extras
  2. Create subdirectories for each set of additional drivers based on the stage library name as follows: 
    /opt/sdc-extras/<stage library name>/lib/
    For example, to install drivers for JDBC, create the following subdirectory:
    /opt/sdc-extras/streamsets-datacollector-jdbc-lib/lib/
    To also install drivers for JMS, create the following subdirectory: 
    /opt/sdc-extras/streamsets-datacollector-jms-lib/lib/
  3. Copy the drivers to the appropriate subdirectories.
  4. In the Data Collector environment configuration file, sdc-env.sh or sdcd-env.sh located in the $SDC_DIST/libexec directory, add the STREAMSETS_LIBRARIES_EXTRA_DIR environment variable and point it to the driver directory, as follows:
    export STREAMSETS_LIBRARIES_EXTRA_DIR="<driver directory>"

    For example: 

    export STREAMSETS_LIBRARIES_EXTRA_DIR="/opt/sdc-extras/"
  5. When using the Java Security Manager, which is enabled by default, update the Data Collector security policy to include the driver directory as follows:
    • In the Data Collector configuration directory, edit the security policy:
      $SDC_CONF/sdc-security.policy
    • Add the following lines to the file: 
      // user-defined additional driver directory
grant codebase "file://<driver directory>-" {
  permission java.security.AllPermission;
};
    For example: 
    // user-defined additional driver directory
grant codebase "file:///opt/sdc-extras/-" {
  permission java.security.AllPermission;
};
  6. Restart Data Collector.

The drivers in the directory become available to their respective stages.
