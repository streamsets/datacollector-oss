Installation and Configuration

You can install the Data Collector as a tarball or service. After installation, you can create additional users and customize the Data Collector.

Installation Requirements

Install the Data Collector on a machine with the following minimum requirements:

Requirement Description
Operating Systems Use one of the following operating systems and versions:
  • Mac OS X
  • Linux CentOS 6 or 7
  • RedHat Enterprise Linux 6 or 7
  • Ubuntu 14.04
Java Java SE 7
Browser Use the latest version of one of the following:
  • Chrome
  • Firefox
  • Safari

Tarball Installation and Launch

Install the Data Collector on the machine that you want to run a pipeline. After installation, you can launch the Data Collector.

  1. Download StreamSets SDC tarball and extract it to desired location.
  2. Optionally, set environment variables to point to appropriate directories:
    • SDC_CONF - SDC configuration directory.
    • SDC_DATA - SDC data directory, where pipeline state and configuration are stored.
    • SDC_LOG - SDC logging directory.
  3. Use the following command to launch the Data Collector and run it in the background: 
    $SDC_HOME/bin/sdc &

Service Installation and Launch

Install the Data Collector on the machine that you want to run a pipeline. After installation, you can launch the Data Collector.

Installing the Data Collector as a service requires sudo privileges on the root directory.

This procedure walks through setting the default directories, but you can alternatively use the libexec/sdcd-env.sh script.
  1. Download StreamSets SDC tarball and extract it to desired location.
  2. Copy initd/_sdcinitd_prototype to the /etc/init.d/ directory as sdc.
  3. Edit the sdc file and set the SDC_DIST environment variable to the location where you extracted the SDC tarball.
  4. Use the following command to make the sdc file executable: 
     sudo chmod 644 /etc/init.d/sdc
  5. Create a system user and group to run the Data Collector.
    The default user and group are sdc/sdc.
  6. Create the /etc/sdc directory and copy all files from etc into this directory.
  7. Create the SDC log directory at /var/log/sdc and change the owner to sdc:sdc.
  8. Create the SDC data directory at /var/run/sdc and change the owner to sdc:sdc.
  9. To launch the Data Collector, use the following command: 
    /etc/init.d/sdc start

Creating Additional Users

The Data Collector includes a default login, admin/admin. You can create additional users for increased security.

You can use md5 to encrypt login information.

  1. Modify the local-realm.properties file located in the etc directory.
    When installed as a service, the location is /etc/sdc/local-realm.properties.
  2. For each new user, add a line using the following format: 
     <username>: <password>[,<rolename> â€¦]
    Currently, only the user role is supported.
  3. To use a password that is not stored in plain text, use the following format for the password: 
    MD5:<password-hash>
  4. You can generate the hash by running an md5 program, such as md5 on Mac OS X or md5sum on Linux program.
    For example: 
    echo -n "<password>" | md5sum
  5. To use digest authentication, hash the combination of <username>:<realm>:<password>.
    For example: 
    echo -n "<username>:<realm>:<password>" | md5sum
  6. Restart the Data Collector to make the new users available.

Customizing the Data Collector

You can customize the Data Collector by configuring the Data Collector properties file.

Use the Data Collector properties file to define properties such as the host name and port number, and SMTP information for sending email alerts. Use a text editor to edit the file. After you edit the file, restart the Data Collector to enable the changes.

The Data Collector properties file, sdc.properties, is located in the following directory:
<DataCollector installdir>/etc/sdc.properties
The Data Collector properties file includes the following general properties:
General Property Description
http.port Port number to use for the Data Collector console.

Default is 18630.
http.authentication HTTP authentication. Use none, digest, or form.

Default is form.
http.digest.realm Realm used by digest authentication.

Default is local-realm.
preview.maxBatchSize Maximum number of records used to preview data.

Default is 10.
preview.maxBatches Maximum number of batches used to preview data.

Default is 10.
production.maxBatchSize Maximum number of records included in a batch when the pipeline runs.

Default is 1000.
production.maxErrorFileSize Maximum error file size.

Default is 1024 MB.

production.maxBackupIndex Maximum number of error files the pipeline can create. When the error file count exceeds this number, older files are removed.

Enter a positive integer. If you enter 0, no additional files are created and the error file is truncated when it reaches the maximum size.

Default is 5.
production.maxErrorRecordsPerStage Maximum number of error records to save in memory for each stage to display during monitoring. When the limit is reached, older error records are discarded.

Default is 100.

production.maxPipelineErrors Maximum number of pipeline errors to save in memory to display during monitoring. When the limit is reached, older errors are discarded.

Default is 100.

max.logtail.concurrent.requests Maximum number of external processes allowed to access the Data Collector log file at the same time through REST API calls.

Default is 5.
ui.help.base.url Base URL for the local help project.

Do not change this value.
ui.online.help.base.url Base URL for the help project hosted at the StreamSets website.

Do not change this value.
ui.refresh.interval.ms Interval in milliseconds that the Data Collector waits before refreshing the Data Collector console.

Default is 2000.
ui.jvmMetrics.refresh.interval.ms Interval in milliseconds that the JVM metrics are refreshed.

Default is 4000.
The Data Collector properties file also includes the following properties for sending email alerts:
Email Alert Properties Description
mail.transport.protocol Use smtp or smtps.

Default is smtp.
mail.smtp.host SMTP host name.

Default is localhost.
mail.smtp.port SMTP port number.

Default is 25.
mail.smtp.auth Whether the SMTP host uses authentication. Use true or false.

Default is false.
mail.smtp.starttls.enable Whether the SMTP host uses STARTTLS encryption. Use true or false.

Default is false.

mail.smtps.host SMTPS host name.

Default is localhost.
mail.smtps.port SMTPS port number.

Default is 25.
mail.smtps.auth Whether the SMTPS host uses authentication. Use true or false.

Default is false.
xmail.username User name for the email account to send alert email.
xmail.password Password for the email account to send alert email.
xmail.from.address Email address to send alert email.
