JavaScript Evaluator

The JavaScript Evaluator processes data based on JavaScript code. Use the JavaScript processor to use JavaScript code to perform custom processing.

The JavaScript Evaluator can process data one record at a time or one batch at a time.

When you use a JavaScript Evaluator in a pipeline, the Data Collector passes a batch of data to the processor and converts the data to a scripting-friendly data structure for processing.

The JavaScript Evaluator supports the following versions of Java:
  • Java 7 - Supports ECMAScript 5 and partially supports JavaScript 1.8. Runs on the Rhino 1.7R3 JavaScript engine.
  • Java 8 - Supports ECMAScript 5.1. Runs on the Nashorn JavaScript engine.
Note: Some Java 8 JavaScript functionality is not available in Java 7 JavaScript. To ensure the JavaScript Evaluator performs as expected, use the same version of Java in development and production.

Processing Mode

You can choose the processing mode to use with the JavaScript Evaluator. You can use the same script in each processing mode. However, you should include error handling in the script before you run in batch mode.

The JavaScript Evaluator provides the following processing modes:

Record by Record
The processor calls the script for each record. The processor passes the record to the script as a map and processes each record individually.
The script does not require error handling logic. Error records are passed to the processor for error handling. The processor handles error records based on the On Record Error property.
Use this mode to avoid including error handling logic in the code. Since this mode calls the script for each record, pipeline performance might be negatively affected.
Batch by Batch
The processor calls the script for each batch. The processor passes the batch to the script as an array and processes the batch at one time.
Include error handling logic in the script. Without custom error handling logic, a single error record sends the entire batch to the processor for error handling. The processor handles error records based on the On Record Error property.
Use this mode to improve performance by processing a batch of data at a time.

JavaScript Scripting Objects

You can use the following scripting objects in the JavaScript Evaluator:
records array
An array of records to process. The records object includes different elements based on the processing mode that you use:
  • Record by Record - The records array includes one record element.
  • Batch by Batch - The records array includes all records in the batch.
record
An object that represents a single record. Includes a single value element. The value element contains the data for the record.
Map and list record fields are mapped to to JavaScript maps and arrays, respectively.
out
An object that writes the record to the output batch. Includes a write(records) element.
err
An object that passes error records to the processor for error handling. Includes a write(records) element with an error message parameter.

Type Handling

Though JavaScript does not use type information when processing data, passing data to the rest of the pipeline requires data types. Note the following type information before you configure the remainder of the pipeline:

Null values convert to String
Regardless of the original data type of a field, when a null value is processed by the JavaScript Evaluator it returns to the pipeline as a string. Because JavaScript does not associate nulls with a data type, the processor uses the String data type.
To avoid type errors in downstream processing, you might use a Type Converter to ensure that fields have data of the appropriate type.
For Java 7, note the following information:
  • Non-null values that are not modified in the processor retain their original type.
  • Numeric data that is modified becames a Double.
  • Other modified data retains their original type.
For Java 8, non-null values retain their original type
Non-null values retain their original type regardless of whether the processor modifies the value.

Configuring a JavaScript Evaluator

Configure a JavaScript Evaluator to use custom JavaScript in a pipeline.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    Required Fields Fields that must include data to be passed to the stage.
    Tip: You might include fields that the stage uses.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the JavaScript tab, configure the following properties:
    JavaScript Property Description
    Record Processing Mode Determines how the JavaScript Evaluator processes data:
    • Record by Record - Processes records individually. Performs error handling.
    • Record Batch - Processes records in batches. Requires error handling code in the script.
    Script JavaScript code to use in the processor.
